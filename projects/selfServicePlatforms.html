Client

Perceptyx

When?

2022

The project

Perceptyx used to have several independent applications that provided their clients with tools related to employee surveys, people analytics, and leadership coaching. The goal of this project was to unify these platforms under a landing page.

Another big part of this project involved start moving clients from always depending on employees to create the projects, to being able to perform some actions by themselves.

At the same time, I was working on a new platform that was 100% self-service.

My favorite part about this project was that I was able to create a user testing process for the company that didn't exist before; after this project this process was aquired by all the platforms and we became user advocates.

The team

I worked on this project with a team that consisted of a scrum master, two PMs, three BE engineers, four FE engineers and a QA analyst.

Goals

  • Create a single log in for all applications that included a landing page where users could find highlights and quick access to all their other applications.

  • Generate a flow between all platforms with a smooth and simple user experience.

  • Design a new platform that allows users to create always-on surveys to gather information about employees sentiment and compare them to their behavior to get information about important issues such as attrition risk or burnout.

Users

  • This project involved all of Perceptyx platforms, so it's target user group were all of their clients.

  • Users were from all genders, ages and countries.

  • These services are used by HR teams, HR leaders, employees, leaders, managers, managers of managers and CEOs.

Process

Create a single log in for all applications that included a landing page where users could find highlights and quick access to all their other applications.

We were able to follow a complete discovery process for this project. We started by creating user personas that were our base to determine the needed features and interactions. We created an empathy map for each one of them so we could understand better their needs. Then we proceeded to create user journeys for all of them.

With all this information gathered during the Discovery phase, we started creating low-fi wireframes.

We conducted interviews to user test these wireframes. We were able to find some issues with our flow so we adjusted them.

We created the first iteration of high-fi designs based on the low-fi wireframes and the feedback from the user testing process. We continued iterating on these designs, validating them with the developers and receiving feedback from our stake holders. We conducted a second round of user interviews, made some adjustments from their feedback and then our developers started working on the code.
